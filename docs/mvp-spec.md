# Clean Crypto MVP — 검증 폼 & 분석 설계서

## 프로덕트 프레이밍

**우리는 특정 리딩방을 평가하는 도구가 아니다.**
유저가 자신의 투자 리딩 환경에서 경험한 패턴을 분석하고, 위험 신호를 인식하도록 돕는 **자가진단 도구**다.

- 방 이름, 운영자 핸들, 초대 링크 등 식별 정보는 수집하지 않음
- 리포트는 "이 방은 위험하다"가 아닌 "입력하신 패턴에서 이런 위험 신호가 확인됩니다"
- 법적 리스크(사실적시 명예훼손) 회피 + 유저 심리적 저항 감소

---

## 유저 플로우

```
랜딩 페이지
  → "내 리딩 환경 진단하기" CTA
  → Stage 1: 리딩 환경 기본 정보
  → Stage 2: 수수료 구조
  → Stage 3: 시그널 품질 & 성과 주장
  → Stage 4: 행동 패턴 & 커뮤니티
  → Stage 5: 내 경험 & 자가 평가
  → 분석 로딩 (LLM 처리)
  → 리포트 페이지
  → 공유 CTA
```

## Stage별 설계

### Stage 1 — 리딩 환경 (2분)

**목적**: 구조적 메타데이터 수집. 방의 기본 프로필만으로도 리스크 윤곽이 잡힌다.

| 입력 항목 | 형식 | 분석 가치 |
|----------|------|----------|
| 운영 플랫폼 | 단일 선택: 텔레그램 / 카카오톡 / 디스코드 / 기타 | 플랫폼별 사기 패턴이 다름. 텔레그램은 익명성 높아 리스크 상승 |
| 운영 기간 | 선택: 3개월 미만 / 3~6개월 / 6~12개월 / 1~2년 / 2년 이상 | 사기 리딩방은 12개월 이상 생존하기 어려움 |
| 대략적 멤버 수 | 선택: 50명 미만 / 50~200 / 200~500 / 500~2,000 / 2,000+ | 대규모 + 고액 수수료 = 비현실적 수익 구조 가능성 |
| 어떻게 알게 됐는지 | 복수 선택: 지인 추천 / SNS 광고 / 유튜브 / 인플루언서 / 검색 / 기타 | 유료 광고 채널 유입은 사기 확률과 상관관계 있음 |

**이 단계가 유저에게 보여주는 것**: 없음 (입력만). 다만 "감사 시작됨" 진행 표시.

---

### Stage 2 — 수수료 구조 (2분)

**목적**: 돈의 흐름은 운영자의 인센티브 구조를 드러낸다. 정당한 운영자는 성과로 유지하고, 사기 운영자는 구조로 추출한다.

| 입력 항목 | 형식 | 분석 가치 |
|----------|------|----------|
| 월 수수료 | 선택: 무료 / 30만원 미만 / 30~100만원 / 100~300만원 / 300만원 이상 | 절대 가격 수준. 고액 + 검증 불가 트랙레코드 = 레드플래그 |
| 등급 구조 | 선택: 단일 등급 / 2~3단계 / VIP/프리미엄 별도 / 없음(무료) | 기본방→VIP 가격 5배 이상 차이 = 기본방이 의도적으로 불만족스럽게 설계됐을 가능성 |
| 상위 등급과 하위 등급의 차이 | 자유 텍스트 | "핵심 콜은 VIP에서만" 같은 모호한 차별화 = 업셀 목적 |
| 무료 체험 또는 환불 보장 | 선택: 있음 / 조건부 / 없음 | 체험 없음 + 고액 = 고위험. 정당한 운영자는 체험을 제공할 수 있음 |
| 장기 결제 유도 여부 | 선택: 없음 / 3개월 할인 / 6개월 할인 / 평생 이용권 | 장기 락인 강하게 푸시 = 이탈 방지 목적 |
| 추천인 리워드 | 선택: 없음 / 소액 할인 / 의미 있는 보상 / 다단계 구조 | 추천 리워드가 수익의 핵심이면 시그널이 아닌 모집이 비즈니스 |
| 수수료 안내 캡처 | 이미지 업로드 (선택) | LLM이 긴급성 언어("마감 임박", "한정 인원"), 비현실적 보장 문구 탐지 |

**이 단계에서 LLM이 분석하는 것**:
- 수수료 × 멤버 수 = 추정 연매출. 개인 시그널 그룹으로 비현실적인 규모인지
- 등급 간 가격 갭의 합리성
- 환불/체험 부재 + 고액의 조합
- 캡처 이미지 내 긴급성/FOMO 문구

---

### Stage 3 — 시그널 품질 & 성과 주장 (3분, 핵심)

**목적**: 제공한다고 주장하는 것이 현실적으로 가능한 것인지, 그리고 그 주장이 검증 가능한지 평가한다.

| 입력 항목 | 형식 | 분석 가치 |
|----------|------|----------|
| 시그널 유형 | 복수 선택: 현물 진입/청산 / 선물(레버리지) / 디파이 / 알트코인 / 매크로 분석 | 선물+고레버리지+초보 대상 = 극고위험 |
| 추천 레버리지 | 선택: 레버리지 없음 / 2~5배 / 5~10배 / 10~20배 / 20배+ | 기본 20배+ 추천 = 레드플래그 |
| 주간 시그널 빈도 | 선택: 1~3개 / 3~7개 / 7~15개 / 15개+ | 너무 많으면 추적 어렵고 체리피킹 쉬움 |
| 주장하는 승률 | 선택: 구체적 수치 안 밝힘 / 60~70% / 70~80% / 80~90% / 90%+ | 크립토에서 80%+ 지속 승률은 통계적으로 불가능에 가까움 |
| 승리/손실 모두 공개 여부 | 선택: 둘 다 공개 / 주로 승리만 / 승리만 / 모름 | 선택적 공개 = #1 조작 기법 |
| 과거 성과 데이터 존재 | 선택: 공개 기록 / 회원에게만 / 구두로만 주장 / 없음 | "구두로만" + 고승률 = 검증 불가 주장 |
| 시그널 캡처 | 이미지 업로드 (1~3장, 선택) | 시그널에 진입가/손절가/목표가/근거가 있는지 판별 |
| 수익 인증 캡처 | 이미지 업로드 (1~3장, 선택) | 체리피킹 여부, 기간 명시 여부, 플랫폼 진위 |

**이 단계에서 LLM이 분석하는 것**:
- **시그널 완성도**: "지금 매수!" vs "BTC 현물 진입 $64,200, TP $67,500, SL $62,800, R:R 2.3:1" — 전자는 분석이 아닌 하이프
- **승률 플라우저빌리티**: 92% 승률 주장 = "이 수치는 글로벌 상위 0.1% 헤지펀드 수준이며, 검증 없이는 통계적으로 불가능합니다"
- **수익 인증 캡처 분석**: 손실이 빠져있는지, 특정 기간만 보여주는지, 레버리지 왜곡이 있는지
- **레버리지 위험도**: 추천 레버리지 × 대상 투자자 수준 → 위험 평가

### Stage 4 — 행동 패턴 & 커뮤니티 (3분, 가장 진단적)

**목적**: "무엇을 말하는가"가 아니라 "어떻게 행동하는가". 행동 시그널은 조작이 가장 어렵다.

| 입력 항목 | 형식 | 분석 가치 |
|----------|------|----------|
| 손실 발생 시 대응 | 선택: 복기 및 리뷰 / 간단히 언급 / 무시 / 시장 탓 / 팔로워 탓 | **가장 강력한 판별 지표**. 정당한 운영자는 손실 복기를 함 |
| FOMO/긴급성 언어 빈도 | 선택: 없음 / 가끔 / 자주 / 주된 영업 방식 | "지금 안 들어오면 손해" = 조작 전술 |
| 비판/의문 제기 허용 여부 | 선택: 자유 토론 / 적당한 중재 / 비판 비권장 / 비판자 밴 | 비판 차단 = 통제 메커니즘 |
| 운영자의 본인 매매 포지션 공개 | 선택: 증거와 함께 공개 / 주장만 / 비공개 / 모름 | Skin in the game = 이해관계 일치 신호 |
| 다른 상품/방 홍보 여부 | 선택: 없음 / 가끔 / 빈번 / 그게 본업인 듯 | 크로스셀링 = 시그널이 진짜 상품이 아닐 수 있음 |
| 커뮤니티 전반적 분위기 | 복수 선택: 분석적 / 교육적 / 하이프 중심 / 공포 기반 / 맹신적 / 서포티브 | 에코챔버 vs 건전한 토론 |
| 커뮤니티 대화 캡처 | 이미지 업로드 (1~3장, 선택) | 실질 토론 vs "감사합니다 방장님" 에코챔버 |
| 손실 처리 방식 캡처 | 이미지 업로드 (1~3장, 선택) | 말과 행동의 불일치 탐지 |

**이 단계에서 LLM이 분석하는 것**:
- **손실 대응 패턴**: 헤지펀드 실사에서 가장 중요한 것 — 손실을 인정하고 복기하는가, 남 탓하는가
- **커뮤니티 역학**: 건전한 질문과 토론이 이루어지는가, 맹목적 추종인가
- **캡처 기반 교차 검증**: 유저가 "비판 비권장"으로 응답했는데 캡처에서는 비판자 밴이 보이면 → 심각도 상향
- **FOMO 전술 강도**: 긴급성 언어가 영업의 핵심인 경우 레드플래그

---

### Stage 5 — 내 경험 & 자가 평가 (2분)

**목적**: 유저 자신의 경험 데이터 + 분석 전 자가 평가 (리포트와 비교하여 인사이트 모먼트 생성).

| 입력 항목 | 형식 | 분석 가치 |
|----------|------|----------|
| 이용 기간 | 선택: 가입 고려 중 / 1개월 미만 / 1~3개월 / 3~6개월 / 6개월+ | 기간별 관점 차이 반영 |
| 시그널 따라한 결과 | 선택: 큰 수익 / 소폭 수익 / 보합 / 소폭 손실 / 큰 손실 / 아직 안 따라함 | 유저 경험 데이터 (바이어스 인지하에 활용) |
| 가장 큰 우려사항 | 자유 텍스트 (필수, 최소 20자) | 유저가 느낀 직감 — 구조적 질문이 놓친 것을 잡아냄 |
| 추가로 알려줄 것 | 자유 텍스트 (선택) | 캐치올 |
| 추가 캡처 | 이미지 업로드 (최대 5장, 선택) | 최종 증거 수집 |
| **분석 전 신뢰도 자가 평가** | **슬라이더 1~10** | **"당신은 7/10으로 평가했지만, 분석 결과 3.2/10" → 강력한 인사이트 모먼트** |

## 분석 엔진

### 4영역 가중 스코어링

| 영역 | 가중치 | 입력 소스 |
|------|--------|----------|
| 구조 (Structure) | 20% | Stage 1 + Stage 2 |
| 성과 (Performance) | 30% | Stage 3 |
| 행동 (Behavior) | 30% | Stage 4 |
| 사회 (Social) | 20% | Stage 1 일부 + Stage 2 일부 + Stage 4 일부 |

### 각 영역 시그널 평가

각 시그널은 3단계:
- **양호 (1.0)**: 우려 없음
- **주의 (0.5)**: 중간 수준 우려
- **위험 (0.0)**: 심각한 우려

영역 점수 = 해당 시그널 평균 × 100

### 크리티컬 오버라이드 (자동 상한)

특정 레드플래그 조합은 다른 영역이 아무리 좋아도 점수를 캡핑:

| 조건 | 최대 점수 |
|------|----------|
| 승률 85%+ 주장 + 검증 불가 트랙레코드 | 35 |
| 환불 없음 + 체험 없음 + 월 200만원+ | 45 |
| 비판자 밴 + 손실 무시/삭제 | 40 |
| MLM급 추천인 구조 | 30 |
| 기본 20배+ 레버리지 추천 | 35 |

### 미입력 처리

점수를 깎는 게 아니라 **신뢰도(Confidence)**를 낮춤:
- "진단 점수: 62/100 (분석 신뢰도: 보통 — 5개 영역 중 3개 완전 응답)"
- 핵심 영역(성과, 행동) 대부분 미응답 시 → "신뢰할 수 있는 분석을 위한 데이터가 부족합니다"
- 이미지 업로드는 신뢰도를 유의미하게 상승시킴

### 모순 처리

모순은 평균 내지 않고 **발견 사항으로 표면화**:
- "수수료 구조는 양호하나, 행동 패턴에서 우려가 확인됩니다"
- 구조적 질문 응답과 캡처 이미지 간 불일치 → 하이라이트

## 리포트 구조

### 1. 진단 요약 (3~4줄)

종합 점수 + 핵심 발견 + 한 줄 결론.
유저의 자가 평가와 분석 점수의 차이를 명시: "당신은 7/10으로 평가하셨으나, 분석 결과 3.8/10으로 평가됩니다."

### 2. 영역별 분석

4개 영역 각각:
- 점수 + 시각적 바
- 2~3개 구체적 발견 사항 (유저 입력 근거 인용)
- 심각도 태그 (위험/주의/참고/양호)

### 3. 위험 신호 목록

심각도 순 정렬. 각 항목:
- 무엇이 탐지됐는지
- 왜 이것이 위험한지
- 정상적인 환경에서는 어떻게 다른지

### 4. 이미지 분석 결과

업로드된 캡처별:
- 구체적 관찰 사항
- 우려 영역 하이라이트
- 모범 사례와의 비교

### 5. 비교 맥락

특정 방을 지목하지 않으면서 맥락 제공:
- "이 수수료 수준의 리딩 환경에서 일반적으로 기대할 수 있는 것은..."
- "주장하신 승률 92%는 글로벌 전문 운용사 상위 0.1% 수준이며..."

### 6. 권고사항

점수대별 구체적 행동 가이드:
- **< 30점**: "이 환경에서의 투자를 재고하시길 권고합니다" + 구체적 이유
- **30~60점**: "추가 확인이 필요합니다" + 확인해볼 구체적 질문 목록
- **60점 이상**: "비교적 양호하나 다음 사항을 확인하세요" + 남은 우려

### 7. 방법론 공개

분석 방법, 사용 데이터, 신뢰도 수준 투명 공개. 도구 자체의 신뢰 구축.

## 기술 구현 범위 (MVP)

### 필수

- 5단계 폼 UI (Next.js 페이지)
- 이미지 업로드 처리 (Vercel Blob 또는 유사)
- LLM 분석 엔진 (OpenAI Vision API 또는 Claude API)
- 리포트 생성 및 렌더링
- 리포트 공유 URL

### 불필요 (MVP 밖)

- 유저 계정/로그인
- 검증 히스토리
- 리딩방 DB

## 법적 고려사항

- 리딩방 식별 정보(이름, 운영자, 링크) 일절 수집하지 않음
- 리포트는 "이 방은 위험하다"가 아닌 "입력하신 패턴에서 위험 신호가 확인됩니다"
- 리포트에 면책 조항: "본 분석은 제공된 정보 기반이며 특정 서비스에 대한 평가가 아닙니다"
- 공유 리포트에 방 식별 정보 불포함
