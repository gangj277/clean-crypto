# 리스크 분석 및 전략적 고려사항

Codex 2라운드 전략 피드백을 기반으로 정리한 핵심 리스크와 대응 방향.

## 핵심 리스크

### 1. Phase 1 — 검증 깊이의 한계

**문제**: 외부에서 수집 가능한 정보만으로는 "깊은 감사"가 어렵다. 실제 거래 데이터(CEX 내부)에 접근할 수 없으므로 선행매매 등을 직접 탐지하기는 어렵다.

**대응**: Phase 1의 목적은 "완벽한 감사"가 아니라 **레드플래그 탐지 + 유저 인식 전환**이다. "수익률 200% 보장" 같은 명백한 사기 징후를 걸러주는 것만으로도 소비자에게 충분한 가치가 있다. 깊은 수준의 검증은 Phase 3 (자체 플랫폼에서 실시간 로깅)에서 해결한다.

### 2. Phase 2 — 역선택 (Adverse Selection)

**문제**: 인증을 가장 원하는 건 실제로 클린한 리딩방이 아니라, 합법적 외피가 필요한 리딩방일 수 있다.

**대응**: 프로모션 대상 선정은 **우리가 능동적으로 컨택**하는 방식이지, 리딩방이 신청하는 방식이 아니다. Phase 1에서 축적된 데이터 + 자체 평가를 기반으로 대상을 선별한다.

### 3. Phase 2 — 이해충돌 가능성

**문제**: 광고주(리딩방) = 평가 대상이 되면, 신용평가사가 평가 대상에게 돈 받는 구조와 같아진다.

**대응**: 
- Phase 1의 클린 체크는 알고리즘 기반으로 자동화, 광고 여부와 무관하게 동작
- 프로모션 리딩방도 동일한 기준으로 지속 평가 → 기준 미달 시 프로모션 중단
- 장기적으로 수익 모델 다변화 (팔로워 구독, 데이터 라이선싱 등)

### 4. Phase 3 — 플랫폼 이전 동기

**문제**: 리딩방 운영자가 텔레그램(무료, 거대 유저베이스)을 떠날 이유가 충분한가?

**대응**: Phase 1~2에서 쌓인 **"이 플랫폼에서 인증받았다"는 브랜드 신뢰**가 이전의 동기가 된다. 추가로, 우리 플랫폼만의 차별화 기능(자동 성과 추적, 리더 보상 인프라 등)이 이전의 실질적 인센티브가 된다.

### 5. 규제 리스크

**문제**: 한국 정부가 리딩방 자체를 전면 금지할 수 있다. 반대로, 규제 플랫폼을 의무화하면 기회가 된다.

**대응**: 
- 규제 강화 시나리오에서는 컴플라이언스 솔루션(B2B)으로 피벗 가능
- 현재 규제 방향(가상자산이용자보호법, 자본시장법 개정)은 "투명성 강화"이므로 우리 방향성과 일치
- 카피트레이딩 등 투자일임업에 해당할 수 있는 기능은 법률 검토 후 도입

## Codex 피드백 중 수용한 인사이트

| 인사이트 | 적용 방식 |
|----------|-----------|
| 비참여 자체가 시그널 | "이 리딩방은 클린 체크 미등록"이라는 정보도 유저에게 가치 있음 |
| 다차원 스코어링 | 단일 점수가 아닌, 여러 지표를 보여주는 대시보드 방식 채택 |
| 시그널 구체성 요구 | 진입가/손절가/목표가 명시 여부를 평가 기준에 포함 |
| 중간급 리더가 비치헤드 | Phase 2에서 평판이 필요한 중간급 리더를 우선 컨택 대상으로 |
| 데이터 축적 기간 필요 | Phase 2 오픈 전 충분한 데이터 임계값 설정 (급하게 오픈하지 않기) |

## Codex 피드백 중 수용하지 않은 것

| 피드백 | 불수용 이유 |
|--------|------------|
| Step 1~2를 건너뛰고 바로 텔레그램 봇(Step 3)부터 | 콜드스타트 문제 과소평가. 소비자향 진입점이 필요 |
| 리더를 먼저 타겟해라 | 우리 프로덕트의 핵심은 **팔로워(소비자)의 불안감 해소**. 공급자가 아닌 수요자부터 |
| 프로덕트 방향을 텔레그램 봇으로 전환 | 봇은 보조 수단이 될 수 있지만, 코어 방향은 소비자향 앱 |
